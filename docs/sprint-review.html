Spec: Suite icon rail

Q1 — Rail on TenantHome?

D3.1 / Icon Rail
The spec recommends the rail appears only on iframe module pages (Field, Service, Cards, Quotes), not on the Shell home page — to avoid duplicating the existing full sidebar. Confirm.

Q2 — Expand on hover or toggle?

D3.1 / Icon Rail
The design handoff shows hover-to-expand (rail widens from 48px to 200px on any hover). This can cause jitter if the user accidentally hovers. An alternative is a click/keyboard toggle to lock the expanded state.

Q3 — Records icon in the rail?

D3.1 / Icon Rail
The current sidebar has a RECORDS section (Customers, Sites, Contacts). Should the icon rail include a "Records" entry that links to the entity browser? Or does Records stay in the TenantHome sidebar only (not in the rail)?

Q4 — Quotes icon for trial tier

D3.1 / Icon Rail
TenantHome already hides Quotes for trial-tier tenants (hideForTier: ['trial']). How should the rail handle trial users?

Q5 — Mobile bottom tab bar

D3.1 / Icon Rail
On mobile (<768px) the rail hides. Without it, mobile users on an iframe module page have no way to switch modules. Options: build a bottom tab bar in D3.3, or defer and leave the existing mobile nav as-is.

High risk. These decisions gate D4.3 build which touches Field (SKS-critical). Read the full spec at eq-shell/docs/specs/d4-2-ia-migration-spec.md before answering.

OQ-1 — Field nav suppression: was v3.5.40 removal intentional?

D4.2 / Field
The audit found Field v3.5.40 removed the shell-mode CSS rules that hid Field's sidebar when embedded in Shell. The Phase 1 fix reinstates those rules. Was the v3.5.40 removal a deliberate product decision, or a temporary measure?

D4.2 / Shell
Shell has two asset surfaces: /:slug/equipment (calibration/service-due focus) and /:slug/data/asset (generic table view). Service also has /assets. All read app_data.assets.

OQ-3 — Assets in the RECORDS sidebar?

D4.2 / Shell
RECORDS in the Shell sidebar shows Customers, Sites, Contacts. There's a separate EQUIPMENT section. Should Assets/Equipment appear under RECORDS, or stay as its own section?

OQ-4 — EQ Quotes sidebar

D4.2 / Quotes
The Quotes Flask app uses ?shell=1 in its iframe URL. The audit couldn't confirm whether Quotes renders a sidebar that's visible in Shell. Does EQ Quotes show its own sidebar/nav when embedded?

OQ-5 — Service /records hub after Phase 2

D4.2 / Service
After Phase 2 (Shell sidebar shows records consistently on all pages), Service's internal /records hub (linking to /customers, /sites, /contacts, /assets) becomes a secondary path. Service already suppresses its sidebar when embedded, so Shell users don't see it. But a user who opens Service in a new tab directly would still see it.

OQ-6 — Phase 1 Field deploy timing

D4.2 / Field / SKS
Phase 1 requires a Field deploy touching auth.js and CSS — load-bearing for SKS in token mode. Shell can ship first (safe), but Field must be deployed carefully with SKS testing.
